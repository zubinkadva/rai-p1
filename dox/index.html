Problem Description

On the table of our R12 robotic arm there exists a cuboid of height 1000 (10 cm), width 700 (7cm) and length 1100 (11cm). Plan the robotic arm to inspect the object from all reachable sides, taking pictures of each reachable 1x1cm2 from a 3 cm distance . (Only the front and top of the object are reachable.) Use the TMOVETO command, and implement an inverse kinematics function to convert these to AJMA commands.

Background

1 cm = 100 robot units

1 degree = 100 robot units

The object is placed at 0 degrees with respect to the 0x and 90 degrees for 0y axes and with the center of the top back side at -4500 0 0. It's position is:
-4500, 550, -1000;
-4500, -550, -1000;
-3800, -550, -1000.

HAND_LENGTH = 1000;

CAPTURE_DISTANCE = 300;

CAPTURE_OFFSET = HAND_LENGTH + CAPTURE_DISTANCE;

CAPTURE_DIAMETER = 100;

CAPTURE_RADIUS = CAPTURE_DIAMETER / 2;

Commands

Typically, the starting position of the arm is straight upward (0 0 5000). You can always return to this position with the command HOME

The robotic arm holds a camera that can take pictures on the command: x CAPTURE., where x is an integer number.

To move the wrist joint of the robot to a position, we use the command:

<HAND> <WRIST> <X> <Y> <Z> TMOVETO


To move the arm to a position by specifying the angle of each joint, we use the command:

<HAND_ANGLE> <WRIST_ANGLE> <ELBOW_ANGLE> <SHOULDER_ANGLE> <WAIST_ANGLE> AJMA


The robot is controlled by sending the above commands either through a web interface or via a GET request using the URL

http://debatedecide.fit.edu/robot.php?o=369&m=Y&p=%lt;PASSWORD>&c=<COMMAND>


After an image is captured, it can be accessed via the URL

http://debatedecide.fit.edu/robot/<IMAGE>.bmp

Methodology

Since the position of the object was known beforehand, for this project we were able to simply create a program that would output the predetermined locations we needed to take images from. Thought was put into making the movement of the arm efficient, and to ensuring that the arm didn't hit the object being scanned. (Two additional command were added to the sequence to ensure this.)

Additionally, we created kinematics and inverse kinematics functions, which will be used in future projects. A description of the derivation of these functions is available here.
